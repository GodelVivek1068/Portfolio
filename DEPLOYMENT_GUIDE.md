# Deployment Guide for Railway

## Overview
This project is now configured for deployment on Railway with:
- Node.js/Express backend
- React frontend with Vite
- PostgreSQL database

## Prerequisites
1. **Railway Account** - Sign up at https://railway.app
2. **GitHub Repository** - Push your code to GitHub
3. **PostgreSQL Database** - Railway will provide this

## Backend Deployment Steps

### 1. Prepare Your Repository
```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/portfolio.git
git push -u origin main
```

### 2. Create Railway Project
- Go to https://railway.app/dashboard
- Click "New Project"
- Select "Deploy from GitHub"
- Select your portfolio repository
- Choose the "backend" directory as the root

### 3. Configure Environment Variables
In Railway dashboard for Backend:
- Set these environment variables:

```
PORT=3000
NODE_ENV=production
DATABASE_URL=[Auto-generated by Railway PostgreSQL]
JWT_SECRET=your-secure-random-secret-32-chars-minimum
CLIENT_URL=https://your-frontend-url.vercel.app
[optional] GITHUB_TOKEN=your-github-token
```

### 4. Configure Database
- Add PostgreSQL plugin to your Railway project
- Railway will automatically populate DATABASE_URL
- Run migrations/schema setup from Railway console:
```bash
psql $DATABASE_URL -f schema.sql
```

### 5. Deploy
- Railway deploys automatically on push to main branch
- Monitor deployment in the Railway dashboard

## Frontend Deployment Steps

### 1. Deploy with Vercel (Recommended)
```bash
cd frontend
npm install -g vercel
vercel
```

Follow prompts and set:
- Root Directory: `frontend`
- Build Command: `npm run build`
- Output Directory: `dist`

### 2. Set Environment Variables in Vercel
In Vercel project settings:
```
VITE_API_URL=https://your-backend-railway-url
```

### 3. Alternative: Deploy to Railway
- Create another Railway service for frontend
- Set build command: `npm run build`
- Set start command: `npm run preview`

## Important Security Notes

⚠️ **BEFORE DEPLOYING:**

1. **Change JWT_SECRET** - Use a strong, random 32+ character string:
   ```bash
   node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
   ```

2. **Change Default Admin Password** - Update `admin123` in server.js:
   ```javascript
   const password = await bcrypt.hash("YOUR_SECURE_PASSWORD", 10);
   ```

3. **Generate New Database Credentials** - Don't use local defaults

4. **Update CORS Origins** - Ensure CLIENT_URL matches your frontend domain

5. **Use HTTPS Only** - All production URLs should use HTTPS

## Troubleshooting

### "DATABASE_URL not found"
- Ensure PostgreSQL plugin is added to Railway project
- Check environment variables are set in Railway dashboard

### "CORS errors"
- Update CLIENT_URL in backend environment variables
- Ensure frontend URL matches exactly

### "Build fails"
- Check that Node version is 14+ (Railway default is fine)
- Ensure all dependencies are in package.json

### "Frontend can't reach backend"
- Update VITE_API_URL to your Railway backend URL
- Redeploy frontend after changing env vars

## URLs After Deployment

Your deployed application will be accessible at:
- **Backend**: `https://your-project-backend.railway.app`
- **Frontend**: `https://your-project.vercel.app` (or your Railway frontend URL)

1. Make changes locally
2. Commit and push to GitHub
3. Railway (and Vercel) auto-deploy on push
4. No manual deployment steps needed
